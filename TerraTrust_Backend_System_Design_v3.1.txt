WHAT CHANGED FROM PREVIOUS VERSION (v2.0)
--------------------------------------------------------------------------------

1. Self-hosted PostgreSQL replaced by Supabase (cloud-hosted, free tier).
   Reason: No server needed. Supabase provides PostgreSQL + PostGIS + Auth +
   File Storage in one free platform. All PostGIS spatial queries work
   identically.

2. Firebase Auth removed. Supabase Auth handles phone OTP.
   Reason: Consolidation to one platform. Same OTP logic, different library.

3. Satellite fusion engine completely rebuilt.
   Old: Single Sentinel-1 VH band only. Single k-factor linear calibration.
   New: 6-layer data stack — NISAR L-band (primary, from June 2026),
   Sentinel-1 C-band (supplementary), Sentinel-2 optical (NDVI, EVI,
   Red-Edge), GEDI canopy height, SRTM topography. XGBoost model replaces
   single k-factor. Speckle filtering added to all SAR data.
   Expected error: Reduced from ~35% to ~5-7%.

4. Tree height source changed.
   Old: AR measurement from farmer (phone anchor method).
   New: NASA GEDI rh98 canopy height from space. AR height used as fallback
   only when no GEDI footprint exists at that location.

5. Sampling zones are satellite-guided.
   Old: Fixed arbitrary zones (N, S, Centre).
   New: Backend runs Sentinel-2 NDVI analysis to identify distinct vegetation
   density zones. Zones placed inside high, medium, low density areas.

6. Pinata replaced by NFT.Storage for IPFS.
   Reason: NFT.Storage is completely free with no file limits. Backed by
   Filecoin Foundation. Files permanently stored even if NFT.Storage closes.

7. Smart contract fixed.
   Fixed: creditAmount multiplication bug (removed * 10**18 from contract).
   Added: retireCredits() burn function for carbon market compliance.
   Added: Double-mint prevention mapping (auditMinted).
   Added: Hardhat config for Polygon Amoy testnet.

8. Planet Labs NICFI data removed.
   Reason: No response received from Planet Labs. Not included in project.

9. S3 file storage removed.
   Reason: Supabase Storage (1GB free) replaces S3 for evidence photos.

10. OCR engine changed from Tesseract to PaddleOCR PP-OCRv5.
    Reason: PaddleOCR with the dedicated Devanagari model achieves significantly
    higher accuracy on printed Marathi text in government documents (F1-Score
    0.938 vs Tesseract's 0.797 on Indic scripts). PaddleOCR also handles
    complex table/form layouts better — important for 7/12 land records.
    It requires no system-level apt-get install (pure pip), and has built-in
    document orientation correction and page unwarping, removing the need for
    manual deskew in OpenCV preprocessing.
    Model used: devanagari_PP-OCRv5_mobile_rec (CPU-compatible, no GPU needed).

2.1 DESIGN PATTERN
------------------
The backend follows a Modular Monolithic Architecture, designed to be
migratable to Microservices when the project scales. It uses an Asynchronous
Event-Driven model for long-running operations.

What "modular monolithic" means in practice: all services (Auth, Land, Fusion,
Blockchain) live in one Python project and one deployment. But they are
structured as separate modules with clean interfaces between them, so any
single module can be extracted to its own service later without rewriting.

What "async event-driven" means: when a farmer submits their audit data, the
API immediately returns a response ("processing started"). The heavy satellite
computation runs in the background via Celery workers. The mobile app polls for
results. This prevents the API from timing out during 60-second satellite jobs.

SECTION 4: DATABASE DESIGN (SUPABASE + POSTGIS)
================================================================================

4.1 SETUP
---------
Run these SQL commands in Supabase SQL Editor before anything else:

  -- Enable PostGIS extensions
  CREATE EXTENSION IF NOT EXISTS postgis;
  CREATE EXTENSION IF NOT EXISTS postgis_topology;
  CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

  -- Verify PostGIS is working
  SELECT PostGIS_Version();

4.2 TABLE: users
-----------------
Purpose: Stores farmer identity, authentication linkage, and blockchain wallet.

  CREATE TABLE users (
      id               UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
      phone_number     VARCHAR(15) UNIQUE NOT NULL,
      full_name        VARCHAR(255) NOT NULL,
      aadhaar_hash     VARCHAR(64),       -- SHA-256 hash, never plain text
      wallet_address   VARCHAR(42),       -- Polygon public address (0x...)
      role             VARCHAR(20) DEFAULT 'FARMER',  -- FARMER, AUDITOR, ADMIN
      kyc_completed    BOOLEAN DEFAULT FALSE,
      created_at       TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
      updated_at       TIMESTAMP WITH TIME ZONE DEFAULT NOW()
  );

  CREATE INDEX idx_users_phone ON users(phone_number);
  CREATE INDEX idx_users_wallet ON users(wallet_address);

4.3 TABLE: land_parcels
-----------------------
Purpose: Stores verified farm land boundaries as spatial geometry.
This is the most critical table — every biomass calculation is clipped to
this polygon.

  CREATE TABLE land_parcels (
      id               UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
      user_id          UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
      farm_name        VARCHAR(255),      -- User-given name e.g. "North Field"
      survey_number    VARCHAR(50) NOT NULL,   -- Extracted by OCR from 7/12
      district         VARCHAR(100) NOT NULL,
      taluka           VARCHAR(100) NOT NULL,
      village          VARCHAR(100) NOT NULL,
      state            VARCHAR(100) NOT NULL DEFAULT 'Maharashtra',
      geom             GEOMETRY(POLYGON, 4326) NOT NULL,  -- Official boundary
      area_hectares    FLOAT,             -- Auto-calculated by PostGIS
      is_verified      BOOLEAN DEFAULT FALSE,
      boundary_source  VARCHAR(50),       -- 'WMS_AUTO', 'SCRAPE', 'MANUAL'
      ocr_owner_name   VARCHAR(255),      -- Name extracted from 7/12 document
      doc_image_url    VARCHAR(500),      -- Supabase Storage URL of 7/12 scan
      lgd_district_code VARCHAR(10),      -- From LGD API
      lgd_taluka_code   VARCHAR(10),
      lgd_village_code  VARCHAR(10),
      gis_code         VARCHAR(50),       -- Constructed for BhuNaksha WMS
      created_at       TIMESTAMP WITH TIME ZONE DEFAULT NOW()
  );

  -- GIST spatial index — required for all PostGIS geometry queries
  CREATE INDEX idx_land_parcels_geom ON land_parcels USING GIST(geom);
  CREATE INDEX idx_land_parcels_user ON land_parcels(user_id);

  -- Trigger: auto-calculate area_hectares whenever geom is inserted/updated
  CREATE OR REPLACE FUNCTION update_land_area()
  RETURNS TRIGGER AS $$
  BEGIN
      NEW.area_hectares := ST_Area(NEW.geom::geography) / 10000;
      RETURN NEW;
  END;
  $$ LANGUAGE plpgsql;

  CREATE TRIGGER trg_land_area
  BEFORE INSERT OR UPDATE ON land_parcels
  FOR EACH ROW EXECUTE FUNCTION update_land_area();

4.4 TABLE: sampling_zones
--------------------------
Purpose: Stores satellite-guided sampling zones generated per audit session.

  CREATE TABLE sampling_zones (
      id               UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
      land_id          UUID NOT NULL REFERENCES land_parcels(id),
      audit_id         UUID,              -- Linked when audit starts
      zone_label       VARCHAR(5) NOT NULL,  -- 'A', 'B', 'C', etc.
      centre_point     GEOMETRY(POINT, 4326) NOT NULL,  -- GPS to walk to

================================================================================
SECTION 6: SATELLITE FUSION ENGINE
================================================================================

6.1 OVERVIEW
------------
The fusion engine is the scientific core of TerraTrust-AR. It takes the AR tree
scan measurements as ground truth and uses them to calibrate a multi-source
satellite model that predicts biomass for every 10m × 10m area of the farm.

It runs as a Celery background task, invoked after the farmer submits scans.
It typically takes 30-90 seconds to complete.

The engine uses 6 satellite data sources in combination. All are free and
available on Google Earth Engine. The XGBoost machine learning model learns
the relationship between the 12 satellite features and actual measured biomass
at the scan locations, then predicts biomass for the entire farm.

6.2 PHASED SAR USAGE
---------------------
Before June 2026:
  Primary SAR: Sentinel-1 C-band (VH + VV bands)
  NISAR: Available for development/testing only (pre-calibration data)
  Reason: NISAR calibrated data release expected June 2026

After June 2026:
  Primary SAR: NISAR L-band (HH + HV bands + ratio)
  Supplementary: Sentinel-1 C-band
  Reason: L-band penetrates full canopy without saturation, giving better
  biomass estimates in high-density tree areas.
  Estimated error improvement: ~35% → ~5-7%

6.3 WOOD DENSITY CONSTANTS (Chave Equation Parameter)
------------------------------------------------------
These values are used in Chave's allometric equation: AGB = 0.0673 *
(wood_density * DBH^2 * Height)^0.976. They are looked up by species name.

  SPECIES_WOOD_DENSITY = {
      "Teak":            0.60,   # Tectona grandis
      "Eucalyptus":      0.55,   # Eucalyptus spp.
      "Neem":            0.56,   # Azadirachta indica
      "Mango":           0.54,   # Mangifera indica
      "Bamboo":          0.70,   # Bambusa spp.
      "Pongamia":        0.67,   # Pongamia pinnata
      "Subabul":         0.56,   # Leucaena leucocephala
      "Casuarina":       0.69,   # Casuarina equisetifolia
      "Indian Rosewood": 0.75,   # Dalbergia sissoo
      "Drumstick":       0.39,   # Moringa oleifera
      "Amla":            0.74,   # Phyllanthus emblica
  }

Any species not in this dictionary should be rejected at the mobile app level
(TF Lite shows "not approved" message).

6.4 COMPLETE FUSION ENGINE CODE
---------------------------------

  # services/fusion_engine.py

  import ee
  import xgboost as xgb
  import numpy as np
  import asf_search as asf
  from typing import List, Dict

  SPECIES_WOOD_DENSITY = { ... }   # From Section 6.3

  def run_fusion(audit_id: str, land_id: str, tree_scans: List[Dict],
                 land_boundary_geojson: Dict, audit_year: int) -> Dict:

      # --- INITIALISE GEE ---
      credentials = ee.ServiceAccountCredentials(
          email=GEE_SERVICE_ACCOUNT_EMAIL,
          key_file=GEE_SERVICE_ACCOUNT_KEY_PATH
      )
      ee.Initialize(credentials)

      start_date = f'{audit_year}-01-01'
      end_date   = f'{audit_year}-12-31'

      region = ee.Geometry.Polygon(land_boundary_geojson['coordinates'])

      # ====================================================================
      # LAYER 1: NISAR L-BAND (Primary SAR — After June 2026)
      # ====================================================================
      # Before June 2026: skip NISAR, use Sentinel-1 as primary
      nisar_available = check_nisar_availability(
          land_boundary_geojson, start_date, end_date
      )

      if nisar_available and audit_year >= 2026:
          nisar_hh, nisar_hv, nisar_ratio = fetch_nisar_data(
              land_boundary_geojson, start_date, end_date
          )
      else:
          nisar_hh = nisar_hv = nisar_ratio = None

      # ====================================================================
      # LAYER 2: SENTINEL-1 C-BAND SAR (Always used, supplementary after 2026)
      # ====================================================================
      s1 = ee.ImageCollection('COPERNICUS/S1_GRD') \
            .filterBounds(region) \
            .filterDate(start_date, end_date) \
            .filter(ee.Filter.eq('instrumentMode', 'IW')) \
            .filter(ee.Filter.listContains('transmitterReceiverPolarisation', 'VH')) \
            .select(['VH', 'VV']) \
            .median()

      # CRITICAL: Apply speckle filter to reduce radar noise
      # Lee speckle filter — 3x3 kernel
      s1_filtered = s1.focal_median(radius=1, kernelType='square', units='pixels')
      s1_vh = s1_filtered.select('VH')
      s1_vv = s1_filtered.select('VV')
      s1_ratio = s1_vh.divide(s1_vv).rename('VH_VV_RATIO')

      # ====================================================================
      # LAYER 3: SENTINEL-2 OPTICAL (Annual cloud-free composite)
      # ====================================================================
      s2 = ee.ImageCollection('COPERNICUS/S2_SR_HARMONIZED') \
            .filterBounds(region) \
            .filterDate(start_date, end_date) \
            .filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', 20)) \
            .median()

      # NDVI: Normalized Difference Vegetation Index
      # Formula: (NIR - Red) / (NIR + Red) — Bands B8, B4
      ndvi = s2.normalizedDifference(['B8', 'B4']).rename('NDVI')

      # EVI: Enhanced Vegetation Index (does not saturate in high biomass)
      # Formula: 2.5 * (NIR - Red) / (NIR + 6*Red - 7.5*Blue + 1)
      evi = s2.expression(
          '2.5 * ((NIR - RED) / (NIR + 6 * RED - 7.5 * BLUE + 1))',
          {'NIR': s2.select('B8'), 'RED': s2.select('B4'), 'BLUE': s2.select('B2')}
      ).rename('EVI')

      # Red-Edge Band B7 — proven to improve accuracy in plantation forests
      red_edge = s2.select('B7').rename('RED_EDGE')

      # ====================================================================
      # LAYER 4: GEDI CANOPY HEIGHT (Where footprints exist)
      # ====================================================================
      # rh98 = height at which 98% of returned laser energy is below
      # = effective canopy top height = tree height from space
      gedi = ee.ImageCollection('LARSE/GEDI/GEDI02_A_002_MONTHLY') \
              .filterBounds(region) \
              .select(['rh98']) \
              .mean() \
              .rename('GEDI_HEIGHT')

      # ====================================================================
      # LAYER 5: SRTM TOPOGRAPHY (Terrain correction)
      # ====================================================================
      # SRTM: Shuttle Radar Topography Mission — global 30m elevation data
      # Used to correct radar signal distortion caused by terrain slope
      srtm = ee.Image('USGS/SRTMGL1_003')
      elevation = srtm.select('elevation').rename('ELEVATION')
      slope = ee.Terrain.slope(srtm).rename('SLOPE')
      # Slope: steeper slope = radar signal distorted, may over/under estimate

      # ====================================================================
      # STACK ALL FEATURES INTO ONE IMAGE
      # ====================================================================
      if nisar_available and nisar_hh is not None:
          feature_stack = ee.Image.cat([
              nisar_hh,         # Feature 1: NISAR HH backscatter
              nisar_hv,         # Feature 2: NISAR HV backscatter
              nisar_ratio,      # Feature 3: NISAR HH/HV ratio
              s1_vh,            # Feature 4: Sentinel-1 VH
              s1_vv,            # Feature 5: Sentinel-1 VV
              s1_ratio,         # Feature 6: S1 VH/VV ratio
              ndvi,             # Feature 7: Sentinel-2 NDVI
              evi,              # Feature 8: Sentinel-2 EVI
              red_edge,         # Feature 9: Sentinel-2 Red-Edge
              gedi,             # Feature 10: GEDI canopy height
              elevation,        # Feature 11: SRTM elevation
              slope             # Feature 12: SRTM slope
          ])
      else:
          # Pre-June 2026: Use Sentinel-1 as primary (9 features)
          feature_stack = ee.Image.cat([
              s1_vh, s1_vv, s1_ratio,
              ndvi, evi, red_edge,
              gedi, elevation, slope
          ])

      # ====================================================================
      # GROUND TRUTH: CHAVE'S ALLOMETRIC EQUATION
      # ====================================================================
      training_points = []
      for scan in tree_scans:
          # Get tree height — prefer GEDI, fall back to AR measurement
          if scan.get('gedi_height_m') and scan['gedi_height_m'] > 0:
              height = scan['gedi_height_m']
              height_source = 'GEDI'
          else:
              height = scan['height_m']
              height_source = 'AR_FALLBACK'

          wood_density = SPECIES_WOOD_DENSITY.get(scan['species'], 0.55)
          dbh = scan['dbh_cm']

          # Chave's Allometric Equation (Chave et al., 2014)
          # AGB = 0.0673 * (rho * D^2 * H)^0.976
          # rho = wood density (g/cm3), D = DBH (cm), H = height (m)
          # Output: AGB in kg
          agb_kg = 0.0673 * (wood_density * (dbh ** 2) * height) ** 0.976

          # Convert to tonnes per hectare for consistency with satellite units
          # Approximate crown area = 10m x 10m = 100 m2 = 0.01 ha
          agb_t_per_ha = (agb_kg / 1000) / 0.01

          training_points.append({
              'lat': scan['gps']['lat'],
              'lng': scan['gps']['lng'],
              'agb_t_per_ha': agb_t_per_ha,
              'height_source': height_source
          })

          # Update scan record with calculated AGB
          update_scan_agb(scan['id'], agb_kg, height_source)

      # ====================================================================
      # TRAIN XGBOOST ON AR GROUND TRUTH + SATELLITE FEATURES
      # ====================================================================
      # Build GEE FeatureCollection with training points + satellite values
      training_fc = build_training_fc(training_points, feature_stack)

      # XGBoost Gradient Boosted Trees (replaces single k-factor approach)
      # Uses GEE's built-in smileGradientTreeBoost classifier
      xgb_model = ee.Classifier.smileGradientTreeBoost(
          numberOfTrees=100,      # 100 decision trees
          shrinkage=0.05,         # Learning rate
          samplingRate=0.7,       # Use 70% of data per tree (prevents overfitting)
          maxNodes=None           # Let tree depth be determined automatically
      ).setOutputMode('REGRESSION').train(
          features=training_fc,
          classProperty='agb_t_per_ha',
          inputProperties=feature_stack.bandNames()
      )

      # ====================================================================
      # PREDICT BIOMASS FOR EVERY PIXEL IN THE LAND BOUNDARY
      # ====================================================================
      # Each pixel = 10m x 10m = 100 m2
      # Predicted value = AGB in tonnes per hectare for that pixel
      biomass_map = feature_stack.classify(xgb_model)

      # Sum all pixel values * pixel area to get total farm biomass
      pixel_area_ha = 0.01   # 100 m2 = 0.01 hectares

      total_biomass_result = biomass_map.multiply(pixel_area_ha).reduceRegion(
          reducer=ee.Reducer.sum(),
          geometry=region,
          scale=10,
          maxPixels=1e9
      ).getInfo()

      total_biomass_tonnes = total_biomass_result.get('classification', 0)

      return {
          'total_biomass_tonnes': total_biomass_tonnes,
          'training_points_count': len(training_points),
          'features_used': 12 if nisar_available else 9,
          'nisar_used': nisar_available
      }


  def fetch_nisar_data(geojson, start_date, end_date):
      """
      Fetch NISAR L-band data from Alaska Satellite Facility.
      Available from October 2025. Fully calibrated from June 2026.
      """
      import asf_search as asf
      from shapely.geometry import shape

      polygon = shape(geojson)
      wkt = polygon.wkt

      results = asf.search(
          platform=asf.PLATFORM.NISAR,
          intersectsWith=wkt,
          processingLevel='GCOV',   # Geocoded Covariance — best for biomass
          start=start_date,
          end=end_date
      )

      if not results:
          return None, None, None

      # Download and process NISAR data
      # Then ingest as GEE image for consistent pixel-level operations
      # This is a simplified representation — actual implementation requires
      # downloading HDF5 files, extracting HH/HV bands, and uploading to GEE
      # as an Earth Engine asset.

      # Once ingested as GEE asset:
      nisar_img = ee.Image(f'users/terratrust/nisar_{start_date[:4]}')
      hh = nisar_img.select('HH').rename('NISAR_HH')
      hv = nisar_img.select('HV').rename('NISAR_HV')
      ratio = hh.divide(hv).rename('NISAR_HH_HV_RATIO')

      return hh, hv, ratio

6.5 CARBON CREDIT CALCULATION
-------------------------------
Run after fusion engine returns total_biomass_tonnes:

  def calculate_credits(total_biomass_tonnes: float,
                         land_id: str,
                         audit_year: int) -> Dict:

      # Query last year's biomass from database
      prev_audit = get_previous_audit(land_id, audit_year - 1)
      prev_biomass = prev_audit['total_biomass_tonnes'] if prev_audit else 0

      delta_biomass = total_biomass_tonnes - prev_biomass

      if delta_biomass <= 0:
          # No growth — no credits earned
          return {
              'credits_issued': 0,
              'reason': 'No biomass growth detected compared to previous year'
          }

      # Carbon fraction of dry biomass = 0.47 (IPCC standard value)
      carbon_tonnes = delta_biomass * 0.47

      # CO2 equivalent: carbon * (44/12) = carbon * 3.667
      co2_equivalent = carbon_tonnes * 3.667

      # 1 CTT = 1 tonne CO2 equivalent
      credits_issued = co2_equivalent

      return {
          'prev_biomass': prev_biomass,
          'current_biomass': total_biomass_tonnes,
          'delta_biomass': delta_biomass,
          'carbon_tonnes': carbon_tonnes,
          'co2_equivalent': co2_equivalent,
          'credits_issued': credits_issued
      }

SECTION 7: BLOCKCHAIN ARCHITECTURE
================================================================================

7.1 WHY BLOCKCHAIN
------------------
Blockchain is used exclusively as a tamper-proof permanent record. Once a
carbon credit is minted on Polygon, the record cannot be edited, deleted, or
disputed. Carbon market buyers can verify every credit's origin, measurement
data, and land ownership proof by reading the on-chain certificate.

7.2 TOKEN STANDARD: ERC-1155
------------------------------
ERC-1155 (Multi-Token Standard) is used because the system needs two types
of tokens per audit:

Type 1 — Carbon Credit Token (Fungible, ID=1)
  Fungible means all credits are interchangeable. 1 CTT from Farm A is worth
  exactly the same as 1 CTT from Farm B. Like currency notes.
  These are the actual tradable carbon credits.

Type 2 — Audit Certificate NFT (Non-Fungible, unique ID per audit)
  Each audit certificate is unique — it represents a specific farm, a specific
  year, with specific measurement data linked via IPFS.
  These prove how and where each batch of credits was generated.

Both are minted in one transaction, saving on blockchain fees.

7.3 WHAT GETS STORED ON IPFS (via NFT.Storage)
------------------------------------------------
Before minting, the backend creates a JSON metadata package containing all
evidence for the audit. This is uploaded to IPFS via NFT.Storage. The
returned CID (permanent content address) is stored on blockchain.

  // SPDX-License-Identifier: MIT
  pragma solidity ^0.8.20;

  import "@openzeppelin/contracts/token/ERC1155/ERC1155.sol";
  import "@openzeppelin/contracts/access/Ownable.sol";

  contract TerraTrustToken is ERC1155, Ownable {

      // ID = 1 is used for all fungible carbon credit tokens
      uint256 public constant CARBON_CREDIT = 1;

      // Stores IPFS evidence link per audit certificate
      // auditId => "ipfs://CID..."
      mapping(uint256 => string) public auditEvidence;

      // Prevents minting the same land+year combination twice
      // keccak256(landId + auditYear) => bool
      mapping(bytes32 => bool) public auditMinted;

      // Tracks how many credits each address has permanently retired
      mapping(address => uint256) public retiredCredits;

      // Emitted every time credits are permanently retired
      // Creates permanent public record for carbon market compliance
      event CreditRetired(
          address indexed retiredBy,
          uint256 amount,
          string retirementReason,
          uint256 timestamp
      );

      constructor() ERC1155("") Ownable(msg.sender) {}

      // ================================================================
      // MINT FUNCTION — called only by backend Oracle wallet
      // ================================================================
      function mintAudit(
          address farmer,        // Farmer's wallet address
          uint256 auditId,       // Unique numeric ID for this audit
          uint256 creditAmount,  // Number of credits e.g. 12 for 12 tonnes
                                 // NO multiplication — send actual number
          string memory ipfsHash, // "ipfs://CID..." — points to evidence
          string memory landId,  // Internal land UUID as string
          uint256 auditYear      // e.g. 2026
      ) public onlyOwner {

          // PREVENT DOUBLE MINTING
          // Same land cannot be minted twice in the same year
          bytes32 auditKey = keccak256(abi.encodePacked(landId, auditYear));
          require(!auditMinted[auditKey],
              "Credits already minted for this land this year");
          auditMinted[auditKey] = true;

          // Mint fungible carbon credit tokens to farmer
          // creditAmount is the actual number of CTT (e.g. 12)
          _mint(farmer, CARBON_CREDIT, creditAmount, "");

          // Mint unique audit certificate NFT to farmer
          _mint(farmer, auditId, 1, "");

          // Store IPFS link permanently on-chain
          auditEvidence[auditId] = ipfsHash;
      }

      // ================================================================
      // RETIRE FUNCTION — called by carbon credit buyer
      // ================================================================
      // When a company buys credits to offset emissions, they must RETIRE
      // (permanently burn) those credits to prove usage. Without this,
      // the same credit could be sold twice. All major carbon markets
      // (Verra, Gold Standard) require on-chain retirement records.
      function retireCredits(
          uint256 amount,
          string memory reason   // "Offset for Q1 2026 manufacturing emissions"
      ) public {

          require(
              balanceOf(msg.sender, CARBON_CREDIT) >= amount,
              "Insufficient credits to retire"
          );

          // Permanently burn — cannot be undone
          _burn(msg.sender, CARBON_CREDIT, amount);

          // Record total retired by this address
          retiredCredits[msg.sender] += amount;

          // Emit permanent public retirement record
          emit CreditRetired(msg.sender, amount, reason, block.timestamp);
      }
  }

  # services/minting_service.py
  import json, requests
  from web3 import Web3
  from eth_account import Account
  import os

  w3 = Web3(Web3.HTTPProvider(os.getenv('ALCHEMY_POLYGON_AMOY_URL')))
  admin_account = Account.from_key(os.getenv('ADMIN_WALLET_PRIVATE_KEY'))

  with open('./contracts/artifacts/TerraToken_ABI.json') as f:
      CONTRACT_ABI = json.load(f)

  contract = w3.eth.contract(
      address=w3.to_checksum_address(os.getenv('CONTRACT_ADDRESS')),
      abi=CONTRACT_ABI
  )

  def upload_to_ipfs(metadata: dict) -> str:
      """Upload audit evidence JSON to NFT.Storage. Returns IPFS CID URL."""
      response = requests.post(
          'https://api.nft.storage/upload',
          headers={
              'Authorization': f'Bearer {os.getenv("NFT_STORAGE_API_KEY")}',
              'Content-Type': 'application/json'
          },
          json=metadata,
          timeout=30
      )
      response.raise_for_status()
      cid = response.json()['value']['cid']
      return f'ipfs://{cid}'

  def mint_carbon_credits(farmer_address: str, audit_id_int: int,
                           credit_amount: int, metadata: dict,
                           land_id: str, audit_year: int) -> Dict:
      """
      Complete minting flow:
      1. Upload evidence to IPFS
      2. Build blockchain transaction
      3. Sign and send transaction
      4. Wait for confirmation
      5. Return tx_hash
      """
      # Step 1: Upload evidence to IPFS
      ipfs_url = upload_to_ipfs(metadata)

      # Step 2: Build transaction
      nonce = w3.eth.get_transaction_count(admin_account.address)
      gas_price = w3.eth.gas_price

      transaction = contract.functions.mintAudit(
          w3.to_checksum_address(farmer_address),
          audit_id_int,
          credit_amount,    # Actual number, no multiplication
          ipfs_url,
          land_id,
          audit_year
      ).build_transaction({
          'from':     admin_account.address,
          'nonce':    nonce,
          'gas':      300000,
          'gasPrice': gas_price,
          'chainId':  80002   # Amoy testnet. Use 137 for mainnet.
      })

      # Step 3: Sign and send
      signed_txn = admin_account.sign_transaction(transaction)
      tx_hash = w3.eth.send_raw_transaction(signed_txn.raw_transaction)

      # Step 4: Wait for confirmation (up to 60 seconds)
      receipt = w3.eth.wait_for_transaction_receipt(tx_hash, timeout=60)

      if receipt['status'] == 0:
          raise Exception(f"Transaction failed: {tx_hash.hex()}")

      return {
          'tx_hash': tx_hash.hex(),
          'ipfs_url': ipfs_url,
          'block_number': receipt['blockNumber'],
          'gas_used': receipt['gasUsed']
      }

  # tasks/fusion_task.py
  from tasks.celery_app import celery_app
  from services.fusion_engine import run_fusion, calculate_credits
  from services.minting_service import mint_carbon_credits
  from app.database import update_audit_status, get_audit_data

  @celery_app.task(bind=True, max_retries=2)
  def run_audit_fusion(self, audit_id: str):
      try:
          # Update status: CALCULATING
          update_audit_status(audit_id, 'CALCULATING')

          # Fetch all data needed for fusion
          audit_data = get_audit_data(audit_id)

          # Run satellite fusion
          fusion_result = run_fusion(
              audit_id=audit_id,
              land_id=audit_data['land_id'],
              tree_scans=audit_data['tree_scans'],
              land_boundary_geojson=audit_data['boundary'],
              audit_year=audit_data['audit_year']
          )

          # Calculate carbon credits from biomass result
          credit_result = calculate_credits(
              total_biomass_tonnes=fusion_result['total_biomass_tonnes'],
              land_id=audit_data['land_id'],
              audit_year=audit_data['audit_year']
          )

          # Save results to database
          save_audit_results(audit_id, fusion_result, credit_result)

          # If credits > 0, trigger minting
          if credit_result['credits_issued'] > 0:
              update_audit_status(audit_id, 'READY_TO_MINT')
              run_minting_task.delay(audit_id)
          else:
              update_audit_status(audit_id, 'COMPLETE_NO_CREDITS')

      except Exception as exc:
          update_audit_status(audit_id, 'FAILED')
          raise self.retry(exc=exc, countdown=30)

9.1 SETUP REQUIREMENTS

First-run note:

  caches them locally. This download happens once only. Subsequent calls use

9.2 PREPROCESSING PIPELINE
----------------------------

Note on preprocessing vs previous version:
  In the Tesseract version, OpenCV was responsible for deskewing (correcting
  tilt) before OCR. With PaddleOCR, deskewing is handled internally by the
  use_doc_orientation_classify and use_doc_unwarping flags. The preprocessing
  pipeline is therefore simpler — only basic image cleanup is done manually.

  # services/ocr_service.py
  import cv2
  import numpy as np
  import re
  from typing import Dict
  from paddleocr import PaddleOCR

  # Initialise PaddleOCR once at module level — not inside the function.
  # Initialising inside the function would reload the model on every request,
  # which is slow. Module-level init loads the model once when the server starts.
  ocr_engine = PaddleOCR(
      text_recognition_model_name="devanagari_PP-OCRv5_mobile_rec",
      use_doc_orientation_classify=True,  # Auto-detect and correct page rotation
      use_doc_unwarping=True,             # Correct page curl and perspective warp
      use_textline_orientation=True,      # Handle upside-down text lines
      device="cpu"                        # Free tier server — no GPU available
  )

  def preprocess_document_image(image_bytes: bytes) -> np.ndarray:
      """
      Prepare 7/12 document image for PaddleOCR.
      Deskew is NOT done here — PaddleOCR handles orientation internally.
      OpenCV is used only for contrast and noise cleanup.
      """
      # Convert bytes to numpy array
      nparr = np.frombuffer(image_bytes, np.uint8)
      img = cv2.imdecode(nparr, cv2.IMREAD_COLOR)

      # Step 1: Convert to greyscale
      grey = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

      # Step 2: Adaptive thresholding (handles uneven lighting and shadows)
      # Government documents are often photographed with uneven phone lighting
      thresh = cv2.adaptiveThreshold(grey, 255,
                                      cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
                                      cv2.THRESH_BINARY, 11, 2)

      # Step 3: Denoise — remove camera noise from low-light photos
      denoised = cv2.fastNlMeansDenoising(thresh, h=10)

      return denoised

  def extract_fields_from_document(image_bytes: bytes) -> Dict:
      """
      Run PaddleOCR and extract structured fields from 7/12 document.
      Returns extracted fields dict or raises ValueError if extraction fails.
      """
      processed = preprocess_document_image(image_bytes)

      # Run PaddleOCR with Devanagari model
      result = ocr_engine.predict(processed)

      # Assemble all recognised text lines into one string
      # result is a list of pages, each page is a list of text line dicts
      full_text = ""
      for page in result:
          for line in page:
              full_text += line["rec_text"] + "\n"

      # Extract fields using regex (handles both English and Marathi labels)
      survey_no = re.search(
          r'(?:Survey|Gat|Gut|सर्व्हे|गट)\s*[Nn]o\.?\s*[:\-]?\s*(\d+(?:/\d+)?)',
          full_text, re.IGNORECASE
      )
      owner = re.search(
          r'(?:Name of Owner|Owner|मालकाचे नाव|धारकाचे नाव)\s*[:\-]?\s*'
          r'([A-Z][a-zA-Z\s]+|[\u0900-\u097F\s]+)',
          full_text, re.IGNORECASE
      )
      village = re.search(
          r'(?:Village|गाव)\s*[:\-]?\s*([A-Z][a-zA-Z\s]+|[\u0900-\u097F\s]+)',
          full_text, re.IGNORECASE
      )
      taluka = re.search(
          r'(?:Taluka|तालुका)\s*[:\-]?\s*([A-Z][a-zA-Z\s]+|[\u0900-\u097F\s]+)',
          full_text, re.IGNORECASE
      )
      district = re.search(
          r'(?:District|जिल्हा)\s*[:\-]?\s*([A-Z][a-zA-Z\s]+|[\u0900-\u097F\s]+)',
          full_text, re.IGNORECASE
      )

      # Check minimum required fields were extracted
      if not survey_no or not owner:
          raise ValueError("Could not extract survey number or owner name. "
                           "Image quality may be too low.")

      # Calculate extraction confidence based on how many fields were found
      fields_found = sum([bool(x) for x in [survey_no, owner, village,
                                              taluka, district]])
      confidence = fields_found / 5

      return {
          'survey_number': survey_no.group(1).strip() if survey_no else None,
          'owner_name':    owner.group(1).strip() if owner else None,
          'village':       village.group(1).strip() if village else None,
          'taluka':        taluka.group(1).strip() if taluka else None,
          'district':      district.group(1).strip() if district else None,
          'state':         'Maharashtra',
          'extraction_confidence': confidence
      }

SECTION 12: SECURITY
================================================================================

Private Key Storage:
  ADMIN_WALLET_PRIVATE_KEY must be stored only in environment variables.
  Never in code files. Never in git. Use Render.com environment variables
  section or a secrets manager in production.

Supabase Row Level Security (RLS):
  Enable RLS on all tables. Each farmer can only read/write their own records.
  Run in Supabase SQL Editor:
    ALTER TABLE land_parcels ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "Users own their land" ON land_parcels
        FOR ALL USING (auth.uid()::text = user_id::text);

    ALTER TABLE ar_tree_scans ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "Users own their scans" ON ar_tree_scans
        FOR ALL USING (land_id IN (
            SELECT id FROM land_parcels WHERE user_id::text = auth.uid()::text
        ));

    ALTER TABLE carbon_audits ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "Users own their audits" ON carbon_audits
        FOR ALL USING (user_id::text = auth.uid()::text);

JWT Validation:
  All endpoints validate Supabase JWT tokens via the Authorization header.
  FastAPI dependency checks token on every protected request:
    async def get_current_user(token: str = Depends(oauth2_scheme)):
        user = supabase.auth.get_user(token)
        if not user: raise HTTPException(401, "Invalid token")
        return user

GEE Credentials:
  Service account JSON file never goes in git.
  In production, set GEE_SERVICE_ACCOUNT_KEY_PATH to a secure mounted volume
  or store the JSON content as an environment variable string and write to
  a temp file on startup.

Smart Contract:
  onlyOwner modifier ensures only the admin wallet can call mint().
  Double-mint mapping prevents duplicate credits per land per year.
  No user-facing function can mint tokens — only the backend Oracle.
