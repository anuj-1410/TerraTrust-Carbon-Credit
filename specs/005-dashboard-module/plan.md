# Implementation Plan: Dashboard Module

**Branch**: `005-dashboard-module` | **Date**: 2026-03-26 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/005-dashboard-module/spec.md`

## Summary

Build the Dashboard module: `HomeScreen` (CTT balance from Polygon blockchain via `services/blockchain.ts`, land parcel list with green/orange/red audit-status badges, 2-entry credit history preview, `credit_earned.json` Lottie when new tokens arrive) and `CreditHistoryScreen` (full audit history list + `react-native-chart-kit` bar chart, year-over-year credits, IPFS/PolygonScan deep links). Update `creditsSlice.ts` to add `lastFetchedAt` and `fetchCreditsThunk`. All screens designed via Stitch MCP first, implemented with NativeWind. Both screens serve MMKV-cached data offline with a staleness badge.

## Technical Context

**Language/Version**: TypeScript 5.0+ strict mode  
**Primary Dependencies**: React Native 0.73, NativeWind 4.0, Redux Toolkit 2.0, ethers.js v6, react-native-chart-kit 6.12, react-native-svg 15, lottie-react-native 7, axios, react-native-config  
**Storage**: MMKV via redux-persist (credits slice вЂ” no blacklist; all fields including `lastFetchedAt` persisted)  
**Testing**: Jest + React Native Testing Library; `fetchCreditsThunk` unit tests; `getLandStatus` utility unit tests  
**Target Platform**: Android (React Native CLI, physical device for end-to-end)  
**Project Type**: Mobile app screen module  
**Performance Goals**: Balance displayed within 3s on 4G; scrollable land list (10+ parcels) smooth at 60 fps  
**Constraints**: Offline-capable via MMKV; no WebView; IPFS links open in device browser only; Blockchain RPC call must have a try/catch fallback  
**Scale/Scope**: 2 screens, 1 slice update, 1 async thunk, 2 Stitch designs
